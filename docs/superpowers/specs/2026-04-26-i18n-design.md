# PetClaw i18n 国际化设计规格

## 1. 目标

为 PetClaw 桌面端引入轻量 i18n 国际化支持，初始支持中文和英文双语，未来可低成本扩展更多语言。翻译资源放在 monorepo 共享包 `petclaw-shared` 中，供 desktop、web、api 各子项目复用。

## 2. 方案选型

对标 LobsterAI 的自研轻量方案，不引入第三方 i18n 库（i18next、react-intl 等）。

**理由：**
- PetClaw 翻译规模约 100 个 key，无需复数/ICU 等高级特性
- 零依赖、完全可控、体积最小
- LobsterAI 已验证此方案在同类 Electron 桌面端可行

## 3. i18n 范围边界

| 纳入 i18n | 不纳入 i18n |
|---|---|
| Renderer UI 文本（按钮、菜单、提示、表单标签） | AI system prompts（managed-prompts.ts / AGENTS.md 模板） |
| 主进程抛给用户的错误提示 | console.log/warn 开发日志 |
| Tray 菜单文本 | 代码注释 |
| Onboarding 引导文字 | |
| Settings 页面所有文案 | |

## 5. 共享包设计（petclaw-shared）

### 5.1 类型定义

```typescript
// petclaw-shared/src/i18n/types.ts
export type Locale = 'zh' | 'en'
export type TranslationKeys = keyof typeof import('./locales/zh')['zh']
```

### 5.2 翻译对象

纯 `Record<string, string>` 扁平结构，key 格式为 `模块.具体项`，不超过两级。支持 `{placeholder}` 插值占位符。

```typescript
// petclaw-shared/src/i18n/locales/zh.ts
export const zh = {
  // 通用
  'common.confirm': '确认',
  'common.cancel': '取消',
  'common.save': '保存',
  'common.delete': '删除',

  // Welcome / Onboarding
  'welcome.morning': '早上好',
  'welcome.afternoon': '下午好',
  'welcome.evening': '晚上好',
  'welcome.tagline': '不止聊天，搞定一切',
  'onboarding.title': '告诉我们关于您',

  // Chat
  'chat.send': '发送',
  'chat.stop': '停止',
  'chat.placeholder': '输入消息...',

  // Settings
  'settings.title': '设置',
  'settings.preferences': '偏好设置',
  'settings.language': '语言',
  'settings.model': '模型管理',

  // Tray
  'tray.show': '显示主窗口',
  'tray.quit': '退出',

  // 错误提示
  'error.dirNotFound': '工作目录不存在：{path}',
  'error.dirDeleted': '该会话的工作目录已不存在：{path}，请创建新会话选择新路径',
  'error.engineStartFailed': '引擎启动失败',
  // ...约 100 个 key
}
```

英文翻译文件 `en.ts` 结构完全一致，key 相同，value 为英文。

### 5.3 导出

```typescript
// petclaw-shared/src/i18n/index.ts
export { zh } from './locales/zh'
export { en } from './locales/en'
export type { Locale, TranslationKeys } from './types'
```

## 6. 主进程 i18n 服务

**文件：** `petclaw-desktop/src/main/i18n.ts`

### 6.1 API

```typescript
initI18n(db: Database.Database): void   // 启动时调用，从 app_config 读或检测系统语言
t(key: string, params?: Record<string, string>): string  // 翻译 + 插值
setLanguage(locale: Locale): void       // 切换语言 + 写入 app_config
getLanguage(): Locale                   // 返回当前语言
```

### 6.2 语言检测逻辑

```
首次启动（app_config 无 language 记录）:
  app.getLocale() → 'zh-CN' / 'zh-TW' → 'zh'
  其余 → 'en'
  写入 app_config

后续启动:
  直接读 app_config.language
```

### 6.3 插值实现

```typescript
function interpolate(template: string, params: Record<string, string>): string {
  return template.replace(/\{(\w+)\}/g, (_, key) => params[key] ?? `{${key}}`)
}
```

## 7. 渲染进程 i18n 服务

**文件：** `petclaw-desktop/src/renderer/src/i18n.ts`

### 7.1 i18nService 单例

```typescript
class I18nService {
  private locale: Locale
  private listeners: Set<() => void>

  t(key: string, params?: Record<string, string>): string
  setLanguage(locale: Locale): void     // 更新 locale + 通知订阅者 + IPC 通知主进程
  getLanguage(): Locale
  subscribe(listener: () => void): () => void  // 返回 unsubscribe 函数
}

export const i18nService = new I18nService()
```

### 7.2 useI18n Hook

```typescript
export function useI18n() {
  const [, forceUpdate] = useReducer(c => c + 1, 0)

  useEffect(() => {
    return i18nService.subscribe(forceUpdate)
  }, [])

  return {
    t: i18nService.t.bind(i18nService),
    language: i18nService.getLanguage()
  }
}
```

组件使用：
```tsx
function SettingsPage() {
  const { t } = useI18n()
  return <h1>{t('settings.title')}</h1>
}
```

### 7.3 初始化

渲染进程启动时通过 IPC `i18n:get-language` 从主进程获取当前语言，初始化 i18nService。

## 8. IPC 通信

| Channel | 方向 | 说明 |
|---|---|---|
| `i18n:get-language` | renderer → main | 启动时获取当前语言 |
| `i18n:set-language` | renderer → main | 用户切换语言，通知主进程持久化 + 更新 Tray |

## 9. 语言切换完整流程

```
用户在设置页切换语言
  → renderer: i18nService.setLanguage('en')
    → 更新内部 locale
    → 通知所有 subscribe 订阅者 → 组件 re-render
    → IPC 'i18n:set-language' 发送到主进程
  → main: setLanguage('en')
    → 写入 app_config 表
    → 更新 Tray 菜单文本
```

## 10. 翻译 key 覆盖范围估算

| 模块 | 预计 key 数 | 说明 |
|---|---|---|
| common | ~10 | 通用按钮、动作词 |
| welcome / onboarding | ~15 | 引导页文案 |
| chat | ~10 | 聊天界面 |
| settings | ~20 | 设置各子页面 |
| sidebar | ~5 | 侧边栏 |
| tray | ~5 | 系统托盘菜单 |
| error | ~15 | 用户可见错误提示 |
| skills / im / cron | ~20 | 各功能模块 |
| **合计** | **~100** | |

## 11. 测试策略

### 11.1 共享包测试

- **翻译完整性**：zh 和 en 的 key 集合必须一致，自动检测漏翻译
- **插值函数**：`{placeholder}` 替换正确性

### 11.2 主进程测试

- initI18n：从 app_config 读语言 / 系统检测 fallback
- setLanguage：写入 app_config
- t()：正确翻译 + 插值 + key 不存在时返回原始 key

### 11.3 渲染进程测试

- useI18n hook：语言切换后触发 re-render
- t() 返回正确翻译文本

## 12. 未来扩展

- **新增语言**：在 `petclaw-shared/src/i18n/locales/` 下新增语言文件（如 `ja.ts`），翻译完整性测试自动覆盖
- **其他子项目**：`petclaw-web`、`petclaw-api` 直接 import `@petclaw/shared` 的翻译对象，各自实现 runtime
- **配置迁移**：后续将 `petclaw-settings.json` 其他字段迁入 `app_config` 表（独立任务，不在 i18n 范围内）
