# I18n 架构设计

## 1. 模块定位

I18n 管理 PetClaw 用户可见文本的语言资源、语言选择、主进程翻译服务和 renderer 翻译服务。PetClaw 使用轻量自研 i18n，不引入 `i18next` 或 `react-intl`。

i18n 是跨包共享契约的一部分：翻译资源、Locale 类型和 helper 放在 `petclaw-shared`；desktop 的主进程和 renderer 各自封装运行时服务。

## 2. 核心概念

- Locale：当前支持 `zh` 和 `en`。
- Translation key：扁平 `模块.键名`。
- shared resources：`petclaw-shared/src/i18n/locales/{zh,en}.ts`。
- main `t()`：主进程用户可见文本翻译。
- renderer `useI18n()`：React UI 翻译入口。
- interpolation：`{placeholder}` 格式插值。

纳入 i18n：

| 纳入 | 说明 |
|---|---|
| Renderer UI 文案 | 按钮、菜单、状态、错误、aria-label、title、placeholder |
| 主进程用户提示 | 菜单、系统状态、用户可见错误摘要 |
| Settings 页面 | 所有标签、说明、错误和操作反馈 |
| 系统菜单 | Dock Menu、Application Menu、Pet Context Menu、非 macOS tray |

不纳入 i18n：

| 不纳入 | 说明 |
|---|---|
| AI system prompts | `managed-prompts.ts`、AGENTS 模板、SYSTEM_PROMPT |
| 开发日志 | `console` / logger 英文日志 |
| 代码注释 | 注释说明代码意图 |
| Provider 官方字段 | 品牌名、模型名、平台官方字段名可保留原文 |

## 3. 总体架构

```text
┌────────────────────────────────────────────────────────────────────┐
│ petclaw-shared                                                     │
│  src/i18n/types.ts                                                  │
│  src/i18n/locales/zh.ts                                             │
│  src/i18n/locales/en.ts                                             │
│  src/i18n/index.ts                                                  │
└──────────────┬──────────────────────────────────┬──────────────────┘
               │                                  │
               ▼                                  ▼
┌──────────────────────────────┐   ┌────────────────────────────────┐
│ Main Process                  │   │ Renderer                       │
│ initI18n(db)                  │   │ i18nService                    │
│ t(key, params)                │   │ useI18n()                      │
│ setLanguage(locale)           │   │ subscribe/re-render            │
└──────────────┬───────────────┘   └───────────────┬────────────────┘
               │ i18n:get/set-language             │
               └───────────────────────────────────┘
```

关键文件：

| 层 | 文件 |
|---|---|
| Shared types/resources | `petclaw-shared/src/i18n/` |
| Main service | `petclaw-desktop/src/main/i18n.ts` |
| Boot IPC | `petclaw-desktop/src/main/ipc/boot-ipc.ts` |
| Renderer service | `petclaw-desktop/src/renderer/src/i18n.ts` |
| Preload | `petclaw-desktop/src/preload/index.ts` |

## 4. 端到端数据流

首次启动语言检测：

```text
app boot
  -> initI18n(db)
  -> read app_config.language
  -> if missing, read app.getLocale()
  -> zh-* maps to zh, others map to en
  -> persist app_config.language
  -> renderer invokes i18n:get-language
  -> i18nService initializes locale
```

用户切换语言：

```text
Settings language select
  -> renderer i18nService.setLanguage(locale)
  -> local subscribers re-render
  -> preload i18n:set-language
  -> main validates locale
  -> persist app_config.language
  -> refresh system menus/tray text
```

用户可见错误：

```text
main service catches error
  -> logs English message with error object
  -> maps to user-facing i18n key
  -> returns structured error / messageKey
  -> renderer t(messageKey)
```

## 5. 状态机与生命周期

```text
unknown
  -> detected
  -> persisted
  -> renderer initialized
  -> changed by user
  -> persisted again
```

约束：

- renderer 初始渲染前必须有默认 locale，不能因为 IPC 未返回导致 UI 空白。
- 主进程语言切换后需要刷新系统菜单文本。
- 翻译资源 key 集合必须中英文完全一致。

## 6. 数据模型

Shared 包结构：

```text
petclaw-shared/src/i18n/
  types.ts
  locales/
    zh.ts
    en.ts
  index.ts
```

类型边界：

```typescript
export type Locale = 'zh' | 'en'
export type TranslationKeys = keyof typeof zh
```

翻译对象使用扁平 `Record<string, string>`：

```typescript
{
  'settings.title': '设置',
  'system.openPetClaw': '打开 PetClaw'
}
```

规则：

- key 使用 `模块.键名`，不超过两级。
- 新增 key 必须中英文同步。
- 同一概念使用同一个 key，不在页面里复制近义 key。
- i18n 资源可以进入 `petclaw-shared`；Electron、React hook、DB 读写不能进入 shared。

## 7. IPC / Preload 契约

语言 IPC 属于 Phase A，可在 BootCheck 前使用：

```text
i18n:get-language
i18n:set-language
```

契约：

- `get-language` 返回当前 `Locale`。
- `set-language` 只接受 `zh` 或 `en`。
- 设置失败时返回用户可见错误，renderer 回滚或展示保存失败状态。
- 新增语言相关 channel 必须同步 `boot-ipc.ts`、`preload/index.ts`、`preload/index.d.ts` 和设置页调用点。

## 8. Renderer 布局、状态与交互

Renderer 只能通过 `useI18n()` 或集中 i18n service 获取文案：

```text
Component
  -> const { t, locale, setLanguage } = useI18n()
  -> render t('module.key')
  -> user changes locale
  -> setLanguage(locale)
```

规则：

- 所有用户可见 UI 文案、状态、错误、aria-label、title、placeholder 必须走 i18n。
- 禁止在组件里硬编码中英文用户文案。
- 阶段性未接入能力的 disabled 文案也必须走 i18n。
- 错误详情可附带原始技术信息，但摘要必须本地化。

## 9. Runtime / Gateway 集成

i18n 不写入 OpenClaw runtime 配置。AI system prompt、AGENTS 模板、skill 文档和 runtime 日志不走 UI i18n。

如果 Gateway 返回需要展示给用户的错误，主进程应将其映射为 PetClaw 自己的用户提示 key；不能把 runtime 原始英文堆栈直接作为 UI 主文案。

## 10. 错误态、安全和权限

插值格式固定为 `{placeholder}`：

```text
t('cron.runFailed', { name: 'Daily summary' })
```

失败策略：

- 找不到 key 时返回原始 key，避免 UI 静默空白。
- 缺失插值参数时保留 `{placeholder}`，便于测试发现。
- 不支持复数、ICU 和嵌套对象；需求超过简单替换时再单独评估。
- 日志仍使用英文，且保留 error 对象作为最后一个参数。

## 11. 与其它模块的关系

| 模块 | 关系 |
|---|---|
| Shared | 存放资源、Locale 类型和 helper |
| Renderer | 使用 `useI18n()` 展示 UI 文案 |
| IPCPreload | 语言查询和设置通过 Phase A IPC |
| SystemIntegration | 系统菜单文案使用主进程 `t()` |
| Settings | 提供语言选择 UI |
| All modules | 用户可见错误和状态必须映射到 i18n key |

## 12. 测试策略

- 中英文 key 集合一致性测试。
- 插值保留和替换测试。
- main `getLanguage/setLanguage` 持久化测试。
- renderer `useI18n()` 切换语言 re-render 测试。
- 系统菜单语言切换刷新测试。
- 用户可见 UI 文案硬编码扫描。
