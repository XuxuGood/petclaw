## 严重问题

### Issue-1: hook:event 双重注册（含 Issue-8）

**位置：**
- `src/main/ipc/pet-ipc.ts`：`hookServer.onEvent` → `send('hook:event')` 给 pet + main 窗口
- `src/main/pet/pet-event-bridge.ts`：`hookServer.onEvent` → `send('hook:event')` 给 pet 窗口 + 状态事件

**现象：** Pet 窗口收到两次 `hook:event`（pet-ipc 发一次，PetEventBridge 再发一次）。

**影响：** MonitorView 渲染重复事件。

**方案：** 删除 `pet-ipc.ts` 整个文件。PetEventBridge 作为 hook 事件的唯一分发入口，构造函数新增 mainWindow 参数，`bindHookEvents` 同时向 pet + main 窗口转发 `hook:event`。同步清理 `ipc/index.ts` 中 `registerPetIpcHandlers` 的调用。一步解决 Issue-1 + Issue-8。

---

### Issue-2 + Issue-3: IM/Scheduler 事件未接入 + activeSessionCount 不归零

**位置：** `src/main/pet/pet-event-bridge.ts`

**现象：**
- `notifyImSessionCreated` / `notifySchedulerTaskFired` 从未被调用
- 即使被调用，`sessionStarted()` 和 CoworkController 的 `message(user)` 会对同一会话重复计数

**影响：** IM/Scheduler 触发时宠物无反应；若强行接线，计数器只增不减导致宠物卡死。

**方案：** 给 `CoworkSession` 模型新增 `origin: 'chat' | 'im' | 'scheduler' | 'hook'` 字段。来源信息持久化在 session 中。删除 `notifyImSessionCreated` / `notifySchedulerTaskFired` 两个方法，删除构造参数 `imGateway` / `cronService`。PetEventBridge 只监听 CoworkController 事件，从 session 的 origin 字段获取来源信息来决定气泡内容。计数器完全由 CoworkController 事件驱动（message(user) +1，complete/error/sessionStopped -1），天然平衡。

**扩展性：** 新增事件源只需加 origin 值；来源特定动画可根据 origin 发不同状态事件；UI 可直接读 origin 展示来源标记；数据库可按 origin 过滤查询。

## 中等问题

### Issue-4: paused 状态下气泡仍然显示

**位置：** `src/renderer/src/pet/PetApp.tsx`

**现象：** `pet:bubble` 监听器没有 `paused` 检查。

**影响：** 宠物暂停时仍弹气泡。

**方案：** 气泡监听器补上 `paused` 门控。暂停时气泡直接丢弃，不缓存（暂停语义 = 宠物停止一切活动）。

### Issue-7: Dragging 状态不应存在

**位置：** `src/renderer/src/pet/state-machine.ts`

**原始问题：** Dragging 结束只回 Idle，中途事件全部丢弃。

**根因分析：** Dragging 状态的存在本身就是设计缺陷。拖拽是视图层行为，不应进入状态机：
- Dragging 播放 `static.webm`，和 Idle 完全相同，无专属动画
- Dragging 屏蔽所有业务事件（AIDone/HookIdle 等），反而造成状态丢失
- 去掉 Dragging 后，拖拽期间保持当前状态（Working 就继续 Working，Sleep 就继续 Sleep），所有事件正常处理

**方案：** 删除 `Dragging` 状态、`DragStart`/`DragEnd` 事件。转移表所有状态中的 `DragStart → Dragging` 行全部删除。PetApp 中 `handleDragMove`/`handleDragEnd` 不再向状态机发事件，拖拽变成纯视图操作。

## 轻微问题

### Issue-9: VideoPlayer 快速切换时 canplay 监听器泄漏

**位置：** `src/renderer/src/pet/PetCanvas.tsx` VideoPlayer 类

**现象：** `playSequence` 只清理 `ended` 监听器，不清理 `canplay` 监听器。快速状态切换时旧 handler 执行导致动画混乱。

**方案：** 用 AbortController 统一管理每轮 `playSequence` 的所有监听器生命周期。每次调用 `playSequence` 时 abort 上一轮，`canplay` 和 `ended` 通过 `{ signal }` 注册，abort 时自动全部清除。删除 `clearEnded()` 方法。
