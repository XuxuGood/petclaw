# 文档同步路由

本文档定义编程完成后的文档同步判断方式。它是 Agent 入口文件的详细路由表；硬性执行规则仍以 [AGENTS.md](../../../AGENTS.md) 和 [CLAUDE.md](../../../CLAUDE.md) 为准。

## 判断原则

开发完成后必须先看 `git diff --name-only`，判断是否改变了架构事实。

需要同步文档的情况：

- 模块职责变化。
- 端到端数据流变化。
- IPC 契约或 preload API 变化。
- 状态模型、数据模型或持久化模型变化。
- 错误边界、安全边界或权限边界变化。
- UI 布局规则、组件规范或视觉规则变化。
- CI/CD、打包、发布或工程工具链流程变化。
- Agent 工作规则、执行入口或必须遵守的流程变化。

不需要同步文档的情况：

- 纯内部实现细节调整，未改变外部契约。
- 局部 bug 修复，未改变数据流、状态模型或错误边界。
- 测试补充，未改变生产行为。
- 文案或样式微调，未改变 UI 规则或布局规范。

如果无需同步，最终回复必须说明原因，例如“仅修复内部实现，未改变 IPC/数据模型/状态边界”。

## 同步步骤

1. 用 `git diff --name-only` 找到候选模块。
2. 按路由表读取候选文档，确认当前实现是否改变架构事实。
3. 需要同步时只更新仍然有效的规则、契约、数据流和边界，不复制过时实现细节。
4. 不需要同步时，在最终回复说明原因。
